# City Guide Framework Documentation

This folder contains the reusable framework documentation for all future city guide projects.

Each city is an independent project that follows the same workflow, structure, and UI system defined here.

---

## Quick Start

Recommended reading order:

1. **`city-project-workflow.md`**
   Understand the complete project lifecycle — from initialization to deployment.

2. **`city-project-template.md`**
   Create a new city project using the standard structure and know which files to reuse vs customize.

3. **`ProjectBrief.md`**
   Define the city's positioning, target users, content scope, and design direction.

4. **`content-schema.md`**
   Understand the JSON data standards and the difference between Guide Layer (Markdown) and Structured Data Layer (JSON).

5. **`ui-navigation-guideline.md`**
   Understand the UI architecture — desktop sidebar, mobile top navigation, section cards, and homepage design.

6. **`content-production-workflow.md`**
   Follow the AI-assisted content production process — from research to validated JSON.

7. **`ai-content-guideline.md`**
   Ensure consistent writing quality, Guide Layer rules, and structured data standards.

```

        ↓
```

---

## Core Principles

- Each city is an **independent project** with its own repository.
- **Reuse the framework** — not the content.
- **Homepage = Essential Guide** — written in Markdown, not a navigation page.
- **Places = Structured Data** — stored in JSON for searchability.
- **Desktop** uses **Sidebar Navigation**.
- **Mobile** uses **Top Navigation** (horizontal, scrollable).
- **Content comes before visual decoration** — no unnecessary animations or hero banners.

---

## Framework Goal

Build a reusable city guide system that can be easily adapted for future cities (Tokyo, Seoul, Taipei, Manila, ...) while maintaining a consistent user experience and a clean, content-first design.

## 核心原则

- 每个城市是一个**独立项目**，拥有自己的仓库。
- **复用框架**，不复用内容。
- **首页 = 出行指南**，用 Markdown 编写，不是导航页。
- **地点 = 结构化数据**，存储在 JSON 中以实现可搜索。
- **桌面端**使用**侧边栏导航**。
- **移动端**使用**顶部导航**（水平滚动）。
- **内容优先于视觉装饰**——无多余动效或大 Banner。

---

## 框架目标

构建一套可复用的城市指南系统，能够轻松适配未来城市（东京、首尔、台北、马尼拉……），同时保持一致的用户体验和简洁、内容优先的设计风格。
