🧠 四、核心设计思想（今天真正完成的）
1. Data-driven architecture
JSON → JS fetch → render cards → UI
2. Multi-page static system（非SPA）
每个分类一个 HTML
无框架
无路由系统
3. GitHub Pages friendly
无 build step
纯静态文件
可直接部署
4. 三层分离结构
层级	内容
UI Layer	UI Design
Behavior Layer	System Spec
Data Layer	JSON Schema

🧪 六、关键问题 & 解决过程
❌ 问题1：sidebar 无法跳转

原因：

路径不一致（/pages/ vs 相对路径）

解决：

统一 pages/*.html 路径结构
❌ 问题2：结构混乱（src vs root）

解决：

删除 src/
转为 GitHub Pages 友好结构
❌ 问题3：fetch JSON 可能错位

解决：

统一 data/ 路径
确保 pages/ 内 JS 使用正确相对路径
